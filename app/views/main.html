Angular Validation provides an effective, performant, highly customizable yet with reliable defaults, validation tool for your Angular web-app.

Concepts:

This module approaches validation via three rule types:

  • Element:
    • Each element is passed to an element specific validator to ensure it will meet its criteria.
  • Group:
    • Elements can belong to groups. Though while in a group elements will stull be subject to their element validators, only once the group passes its validation criteria will it count towards the final form pass/fail.
  • Instance:
    • The instance rule is the term attached to the concept of "Can the user submit the form or not". The instance rule requires all validation criteria, both group and element, to be met.

Installation

Method One:

Clone this repo and copy/paste the files in /build to the directory of your choice in your app. Then reference them in your index.html

Method Two:

bower install ngValidation --save

Of course you still need to reference the files in your index.html

How to Use:

Default Implementation:
  1. Inject the validation module into your App's module declaration: 
    angular.module('MyApp', ['ng-validation'])
  2. Inject the validation module's service into a controller that will be using it: 
    angular.module('MyApp', []).controller('MainCtrl', function($scope, ngValidation) {})
  3. Inside the controller, include: 
    ngValidation.init();
  4. From here, if you wish a form to be validated, add the attribute 
    ng-validation
    to the root of the form/section containing the form fields.
Custom Processing

If you need more than the defaults, such as adding a scenario, follow these steps:

  1. To the same form/section you added 
    ng-validation
    to, add an attribute name: 
    ng-validation="myForm"
    . Please use only alpha-numeric characters.
  2. Once you've added the name, open up the controller where you called 
    ngValidation.init()
    and add your rules as follows: 
    ngValidation.add_rule('myForm', {options}).init();
    make sure the name of the rule is the same name you gave your form in the markup. And of course, dont forget to call 
    .init();
    When you're done adding all the rules you'll need.
  3. You can now give your form custom options:
Available options:
    • Name: rule
    • Type: function
    • Arguments (order matters):
      • Sel: DOM object of the root of the form
      • el: DOM object of the current event target
      • e: current event object
    • Scope (this): The root validation class
    • Description: If you want to completely ignore the default rule for determining whether or not a form is valid, add your own function here.
    • Example:
    • 
myForm: {
  rule: function(sel, el, e) {
    // custom processing here
  }
}
    • Name: exempt
    • Type: array
    • Description: An array containing an indefinite number of indexes of strings that correlate to elements you wish to avoid validation processing. It uses a jQuery like selector syntax, just without the $().
    • Example:
    • 
myForm: {
  // This will ignore all elements with the class of ignore
  exempt: ['.ignore']
}
    • Name: scenarios
    • Type: array of objects
    • Description: An array containing the desired validation scenarios to use.
    • Example:
    • 
myForm: {
  // This will load the scenarios you wish to use (example is using the password scenario)
  scenarios: [{name: 'password', options: {ui: '.meter', acceptance: 'strong'}}]
}

Group 1

Group 2

Standalone Elements

Options:

            
// In the ngValidation injected controller:
ngValidation.add_rule('form_4', {
  groups: [
    {name: 'radios', container: false, required: 'input[name="group1"]', number_required: 1},
    {name: 'checkboxes', container: '#checkboxes', required: 'input[type="checkbox"]', number_required: 1}
  ]
}).init();
